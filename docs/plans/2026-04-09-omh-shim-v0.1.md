## Task 2: Core primitives — errors, dispatch, schema loader, validator

Four small modules. None are more than ~30 lines.

**`omh_shim/errors.py`:**
```python
class ConversionError(Exception):
    """No converter for (source, data_type) or malformed input."""

class ValidationError(Exception):
    """Converter output failed OMH schema validation."""
```

**`omh_shim/_dispatch.py`:**
```python
from collections.abc import Callable
from omh_shim.errors import ConversionError

_Converter = Callable[[dict], dict]
_REGISTRY: dict[tuple[str, str], _Converter] = {}

def register(*, source: str, data_type: str):
    def _wrap(fn): _REGISTRY[(source, data_type)] = fn; return fn
    return _wrap

def lookup(source: str, data_type: str) -> _Converter:
    try:
        return _REGISTRY[(source, data_type)]
    except KeyError as e:
        raise ConversionError(
            f"No converter for source={source!r} data_type={data_type!r}"
        ) from e
```

**`omh_shim/_schema_loader.py`:**
```python
import importlib.resources, json
from functools import lru_cache

@lru_cache(maxsize=None)
def load(schema_id: str) -> dict:
    filename = schema_id.replace(":", "_").replace(".", "-") + ".json"
    with importlib.resources.files("omh_shim.schemas").joinpath(filename).open() as f:
        return json.load(f)
```

**`omh_shim/_validate.py`:**
```python
from jsonschema import Draft7Validator
from omh_shim._schema_loader import load
from omh_shim.errors import ValidationError

def validate_output(output: dict, schema_id: str) -> None:
    errors = sorted(Draft7Validator(load(schema_id)).iter_errors(output),
                    key=lambda e: list(e.absolute_path))
    if errors:
        raise ValidationError(
            f"Output does not conform to {schema_id}: "
            + "; ".join(f"{'/'.join(str(p) for p in e.absolute_path) or '<root>'}: {e.message}"
                        for e in errors)
        )
```

**`omh_shim/__init__.py`** — wire up the public API:
```python
from omh_shim.errors import ConversionError, ValidationError
from omh_shim._dispatch import lookup
from omh_shim._validate import validate_output

__all__ = ["convert", "ConversionError", "ValidationError"]
__version__ = "0.1.0.dev0"

_OMH_SCHEMA_ID = {
    "heart_rate": "omh:heart-rate:2.0",
    "heart_rate_variability": "omh:heart-rate-variability:1.0",
    "step_count": "omh:step-count:3.0",
    "sleep_duration": "omh:sleep-duration:2.0",
    "sleep_episode": "omh:sleep-episode:1.1",
    "physical_activity": "omh:physical-activity:2.0",
}

def convert(source: str, data_type: str, sample: dict, *, validate: bool = True) -> dict:
    output = lookup(source, data_type)(sample)
    if validate:
        validate_output(output, _OMH_SCHEMA_ID[data_type])
    return output

# Source packages register themselves on import.
import omh_shim.sources.ow_normalized  # noqa: E402, F401
import omh_shim.sources.oura_raw       # noqa: E402, F401
```

## Task 4: `ow_normalized` source — all 6 converters in one pass

**Create `omh_shim/sources/ow_normalized/__init__.py`:**
```python
"""Converters for Open Wearables normalized read-API response shapes."""
from omh_shim.sources.ow_normalized import (  # noqa: F401
    heart_rate, heart_rate_variability, step_count,
    sleep_duration, sleep_episode, physical_activity,
)
```

Then write all 6 converter files. They share a pattern: pull fields out of the OW input dict, build an OMH `{header, body}` dict, return it. Every converter is ~20-40 lines.

### Source input shapes (from OW's `backend/app/schemas/responses/activity/`)

- **`TimeSeriesSample`** (heart_rate, heart_rate_variability, step_count): `{timestamp, zone_offset, type, value, unit, source: {source_name, device_model}}`
- **`ActivitySummary`** (step_count rollups, physical_activity, sleep_duration): `{date, source, steps, distance_meters, active_calories_kcal, active_minutes, sedentary_minutes, heart_rate: {avg_bpm, max_bpm, min_bpm}, sleep_total_duration_minutes, sleep_time_in_bed_minutes, ...}`
- **Sleep details object** (sleep_episode): `{record_id, bedtime_start, bedtime_end, sleep_total_duration_minutes, sleep_time_in_bed_minutes, sleep_deep_minutes, sleep_rem_minutes, sleep_light_minutes, sleep_awake_minutes, sleep_stages: [...]}`

### Helper: OMH header

Put this in `omh_shim/sources/_common.py` so all 12 converters share it:

```python
def omh_header(source: str, data_type: str, effective_time: str, device: str) -> dict:
    schema_parts = {
        "heart_rate": ("heart-rate", "2.0"),
        "heart_rate_variability": ("heart-rate-variability", "1.0"),
        "step_count": ("step-count", "3.0"),
        "sleep_duration": ("sleep-duration", "2.0"),
        "sleep_episode": ("sleep-episode", "1.1"),
        "physical_activity": ("physical-activity", "2.0"),
    }[data_type]
    return {
        "id": f"{source}:{data_type}:{effective_time}:{device}",
        "creation_date_time": effective_time,
        "schema_id": {"namespace": "omh", "name": schema_parts[0], "version": schema_parts[1]},
        "acquisition_provenance": {"source_name": device},
    }
```

### The 6 ow_normalized converters

**1. `heart_rate.py`** — input: `TimeSeriesSample` with `type="heart_rate"`. Body: `{heart_rate: {value, unit: "beats/min"}, effective_time_frame: {date_time: timestamp}}`.

**2. `heart_rate_variability.py`** — same input shape with `type="heart_rate_variability"`. Body matches the HRV placeholder schema: `{heart_rate_variability: {value, unit: "ms"}, effective_time_frame: {date_time}}`.

**3. `step_count.py`** — accepts **either** a `TimeSeriesSample` (type=steps) **or** an `ActivitySummary` with `steps` and `date`. Detect which shape: if `type` key present → timeseries sample (1-minute step count at `timestamp`); if `date` key present → daily rollup (time_interval covers the day). Body: `{step_count: int, effective_time_frame: {date_time | time_interval: {start_date_time, end_date_time}}}`.

**4. `sleep_duration.py`** — input: `ActivitySummary` with `sleep_total_duration_minutes` and `date`. Body: `{sleep_duration: {value: minutes*60, unit: "sec"}, effective_time_frame: {time_interval: {start_date_time: midnight, end_date_time: next_midnight}}}`.

**5. `sleep_episode.py`** — input: sleep details object with `bedtime_start`, `bedtime_end`, stage minute counts, optional `sleep_stages` timeline. Body: `{sleep_episode: {time_interval: {start_date_time: bedtime_start, end_date_time: bedtime_end}, total_sleep_time: {value, unit: "sec"}, stages: [{stage, duration: {value, unit: "sec"}} ...]}}`. Stages map: deep→"deep sleep", rem→"REM sleep", light→"light sleep", awake→"awake". **Hardest converter** — if OMH's sleep-episode-1.1 schema has a different stages field name, align to the schema.

**6. `physical_activity.py`** — input: `ActivitySummary` with `steps`, `active_calories_kcal`, `distance_meters`, `active_minutes`, `date`. Body: `{activity_name: "daily activity summary", effective_time_frame: {time_interval: day}, distance: {value, unit: "m"}, kcal_burned: {value, unit: "kcal"}, duration: {value: active_minutes*60, unit: "sec"}, step_count_detail: {step_count: steps}}`. Again, align field names to whatever `omh_physical-activity_2-0.json` actually requires.

**Every converter follows this template:**

```python
"""Convert OW X to omh:Y:V."""
from omh_shim._dispatch import register
from omh_shim.sources._common import omh_header

@register(source="ow_normalized", data_type="<data_type>")
def convert(sample: dict) -> dict:
    # pull fields
    effective_time = sample["..."]
    device = (sample.get("source") or {}).get("device_model") or "unknown"
    # build body
    body = {...}
    return {
        "header": omh_header("ow_normalized", "<data_type>", effective_time, device),
        "body": body,
    }
```

### Reference dicristea's code

Before writing converters, read:
- `data_syn/config/mapping_config.json` — the source-field → OMH-field mapping rules
- `data_syn/utils/record_builder.py` — the actual OMH record construction helpers (especially `build_heart_rate_series`, `build_physical_activity_record`, `build_header`)

The dicristea code is file-based and CLI-driven. We're porting the **logic**, not the code structure. For each converter below, distill the relevant mapping into our pure-function shape.

### Source input shapes (from Oura v2 API)

Oura returns `{"data": [item, ...], "next_token": ...}`. Our converters take **one item**, not the envelope. Caller unwraps `data`.

- **Heart rate** (`/v2/usercollection/heartrate/data[i]`): `{timestamp, bpm, source}`. 1:1 with OW's heart_rate.
- **HRV** (`/v2/usercollection/daily_readiness/data[i]` or detail endpoint): extract HRV field. Daily granularity.
- **Daily activity** (`/v2/usercollection/daily_activity/data[i]`): `{day, steps, active_calories, total_calories, equivalent_walking_distance, low/medium/high_activity_time, ...}`. Produces step_count AND physical_activity records (two separate `convert()` calls with the same input).
- **Daily sleep** (`/v2/usercollection/daily_sleep/data[i]`): `{day, score, contributors: {total_sleep, efficiency, ...}, timestamp}`. Daily sleep summary for `sleep_duration`.
- **Sleep detail** (`/v2/usercollection/sleep/data[i]`): `{bedtime_start, bedtime_end, total_sleep_duration, time_in_bed, deep_sleep_duration, rem_sleep_duration, light_sleep_duration, awake_time, sleep_phase_5_min, ...}`. For `sleep_episode` with structured stages.

### The 6 oura_raw converters

**1. `heart_rate.py`** — trivial 1:1. Field names: `timestamp`, `bpm`. Device hardcoded to `"Oura Ring"` (Oura's heart rate endpoint doesn't carry device metadata).

**2. `heart_rate_variability.py`** — Oura publishes HRV under `daily_readiness`. Extract the HRV field (exact name: check dicristea's `mapping_config.json` — likely `hrv_balance` or similar). Daily granularity, so `effective_time_frame.date_time` is the day's midnight UTC.

**3. `step_count.py`** — Oura `daily_activity.steps`. Time interval spans the day (midnight to next midnight in the day's zone).

**4. `sleep_duration.py`** — Oura `daily_sleep` item. Duration from `contributors.total_sleep` or equivalent. Time interval from `bedtime_start`/`bedtime_end` if present, otherwise the day.

**5. `sleep_episode.py`** — Oura `sleep` detail item. Most complex converter. Map:
- `time_interval` = `bedtime_start` → `bedtime_end`
- `total_sleep_time` = `total_sleep_duration` (seconds)
- `stages` = array built from `deep_sleep_duration`, `rem_sleep_duration`, `light_sleep_duration`, `awake_time` — one entry per stage with aggregate duration. The OMH sleep-episode schema aggregates stages, not per-minute timelines. **Don't parse `sleep_phase_5_min`** unless the schema demands it — summary durations are sufficient.
- See dicristea's `_sleep_interval` and the mapping_config entries for `omh:sleep-episode:1.1`.

**6. `physical_activity.py`** — Oura `daily_activity` item. Same output shape as `ow_normalized.physical_activity`. Field sources: `equivalent_walking_distance` (meters), `active_calories` (kcal), `low_activity_time + medium_activity_time + high_activity_time` summed to `active_minutes`.

## What this plan deliberately skips

- **Per-converter TDD ceremony.** Write the 6 converters in one pass using the first as a template. Tests cover them as a parameterized group. If a test fails, fix it; if it passes, move on.
- **Multiple fixture variants per converter.** One real fixture per converter is enough for v0.1. Edge-case fixtures get added when a real bug surfaces.
- **Separate test files per converter.** Two test files — `test_ow_normalized.py` and `test_oura_raw.py` — parameterized over all 6 data types each. Total test code ~60 lines.
- **Formal "red-green-commit" rhythm per step.** The engineer is capable of writing a converter and its test together and committing when the parameterized suite is green.
- **Over-validation in tests.** `convert()` already validates output by default, so every passing test is implicit schema validation. Don't re-check.
